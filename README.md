# СД онлайн — бот-диагностика + админка

## Что это

- Telegram-бот: приветствие → выбор теста → 10 вопросов с кнопками → результат (🔴/🟡/🟢) → приглашение на Совет директоров
- Админка (веб-страница): база всех прохождений + рассылка сообщений по сегментам

## Что нужно поменять перед запуском

1. **Ссылка на запись (BOOKING_URL)** — в файле `.env` (или в настройках Railway/Render, см. ниже) укажите реальную ссылку на запись на СД
2. **Пароль админки (ADMIN_PASSWORD)** — придумайте свой, не оставляйте стандартный

## Как запустить на хостинге Railway (без программирования)

Railway — сервис, который берёт ваш код и держит его "включённым" 24/7. Понадобится 15-20 минут.

### Шаг 1. Создайте аккаунт
Зайдите на **railway.app**, зарегистрируйтесь (проще всего — через GitHub-аккаунт; если его нет, можно создать за минуту на github.com).

### Шаг 2. Загрузите код
Самый простой способ без командной строки:
1. Создайте новый репозиторий на **github.com** (кнопка "New repository"), назовите, например, `sd-online-bot`
2. На странице репозитория нажмите "uploading an existing file" и перетащите туда все файлы и папки из этого проекта (кроме папки `data`, если она пустая — она создастся сама)
3. Нажмите "Commit changes"

### Шаг 3. Подключите репозиторий к Railway
1. В Railway нажмите **New Project → Deploy from GitHub repo**
2. Выберите ваш репозиторий `sd-online-bot`
3. Railway сам определит, что это Node.js проект, и начнёт сборку

### Шаг 4. Добавьте переменные окружения
В настройках проекта в Railway найдите вкладку **Variables** и добавьте (каждую отдельной строкой, имя = значение):

```
BOT_TOKEN=8867140421:AAFRjFNs5S9pBMGfrEpni7xne-GiebZbvQ4
ADMIN_PASSWORD=ваш_новый_пароль
SESSION_SECRET=любая_случайная_строка_из_букв_и_цифр
BOOKING_URL=https://the-focus.ru/cd_partners01
DATA_DIR=/data
PORT=3000
```

### Шаг 5. Подключите постоянный диск (важно!)
Без этого шага база данных будет стираться при каждом обновлении бота.
1. В проекте Railway найдите раздел **Volumes** (обычно во вкладке сервиса → Settings)
2. Нажмите **New Volume**
3. Укажите путь монтирования: `/data`
4. Сохраните — Railway пересоберёт проект автоматически

### Шаг 6. Получите ссылку на админку
1. Во вкладке **Settings → Networking** нажмите **Generate Domain**
2. Railway выдаст ссылку вида `sd-online-bot-production.up.railway.app`
3. Админка будет доступна по адресу: `https://ваш-домен/admin`

### Шаг 7. Проверьте бота
Откройте своего бота в Telegram (@SD_online_service_bot), нажмите **Start** — должно появиться приветствие и меню.

## Альтернатива: Render.com

Работает почти так же:
1. New → Web Service → подключить GitHub-репозиторий
2. Build command: `npm install`
3. Start command: `npm start`
4. Добавить те же переменные окружения (Environment tab)
5. Добавить Persistent Disk с точкой монтирования `/data` (доступно на платных планах Render — если это критично, Railway с Volume в бесплатном/минимальном тарифе обычно проще для старта)

## Как добавить второй, третий тест в будущем

Откройте файл `src/questions.js` — там внизу есть закомментированный пример. Скопируйте блок `partner_relations` целиком, дайте новый `id`, впишите свои вопросы и тексты результатов. Новый тест сам появится в меню бота — код трогать не нужно.

## Структура проекта

```
sd-bot/
├── index.js           — запуск бота и админки
├── src/
│   ├── bot.js          — логика Telegram-бота
│   ├── admin.js        — веб-админка (база + рассылка)
│   ├── db.js            — хранение данных (JSON-файл)
│   └── questions.js     — ВСЕ вопросы, ответы и тексты (редактировать сюда)
├── data/                — здесь хранится база (создаётся автоматически)
├── .env.example          — пример настроек, скопируйте в .env для локального теста
└── package.json
```

## Локальная проверка (если захотите протестировать на своём компьютере)

Нужен установленный Node.js (nodejs.org, версия 18+).

```
npm install
cp .env.example .env
# откройте .env и впишите свои значения
npm start
```

Админка будет доступна на `http://localhost:3000/admin`.
